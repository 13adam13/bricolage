Template Profile Data

1) Information

Notes: Attach editorial notes to the Template. The Notes will be viewable to other users when you check the Template in to a Desk.
Trail: Lists what Desks the Template has been checked in to (None if it is a new Media).
Asset Type: The type of Template, selected on the Create a New Template page. If it is not associated with an existing Element, the Template's type will be "Generic."
Template Name: The title of the Template. This will be the same as the Asset Type if the new Template is based on an existing Element.
Output Channel: The destination for the Template, once it is deployed.
Category: Where on the site the Template will be deployed.
URI: The address of the Template on your server.
Priority: Tells users how urgent the Template is.
*Note: Priority field must be filled in order to save.
Description: A brief description of the Template.
Expire Date: The date and time you want the Template to be removed.

2) Template Code

Element: The element on which the Template is based, if any.
Data Elements: Elements which automatically appear in the Content section of a Template. Those that indicate "repeatable" can be added multiple times. Those that indicate "required" must be filled out by the user in order to save the Template.
Container Elements: Elements that can be added to a Template, they appear in a pulldown box beneath the Data Elements. Those that indicate "repeatable" can be added multiple times.
Enter additional code for the Template in the field provided. Click Save when finished.
